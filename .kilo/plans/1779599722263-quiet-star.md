# Plan: Simplify CLI/Backend Entry Points & Fix Codex/Fireworks Routing

## Problem Summary

The user ran `cargo run --bin agentflow` and got `"All LLM providers failed. Last error: HTTP request to Anthropic API failed"`. Root cause analysis:

1. **Nexus LLM routing is the actual failure point**: The `FallbackClient::build()` sees `PROXY_URL` is set → enters proxy mode → creates `AnthropicClient` pointing to `http://localhost:8765/v1/messages` (not running) → sees `FIREWORKS_API_KEY` → marks as "external connector" → skips direct API fallbacks → total failure.

2. **Model-aware routing missing**: Registry says `model_backend: "accounts/fireworks/models/kimi-k2p5"`, but `MODEL_PROVIDER_MAP=glm=anthropic,deepseek=openai,gpt=openai` has no `accounts/fireworks=fireworks` mapping, so `resolve_provider_for_model()` returns `None` and the model falls through to the Anthropic proxy.

3. **ForgeNode in main.rs still hardcodes `"claude"`**: The `agentflow.rs` binary already uses `ForgePairNode` (correct), but `binary/src/main.rs` still uses the old `ForgeNode` with hardcoded `"claude"` at line 209.

4. **Confusing .env**: Both proxy AND Fireworks settings are active simultaneously, causing contradictory routing.

5. **Poor error messages**: "HTTP request to Anthropic API failed" doesn't mention the URL or that the proxy isn't running.

## Phase 1: Fix FallbackClient LLM Routing (PRIMARY FIX)

### 1.1 Make `build()` model-aware — check `model_override` before proxy/env

**File**: `crates/agent-client/src/fallback.rs`

Current flow:
```
PROXY_URL set? → proxy chain (AnthropicClient → localhost:8765)
FIREWORKS_API_KEY set? → fireworks chain
Otherwise → direct chain
```

New flow:
```
model_override provided?
  → maps to "fireworks" via MODEL_PROVIDER_MAP? → FireworksClient directly
  → starts with "accounts/fireworks/"? → FireworksClient directly
  → maps to "openai"? → OpenAiClient directly
  → maps to "anthropic" AND no proxy? → AnthropicClient directly
  Otherwise fall through to existing proxy → fireworks → direct chain
```

Key change: **When the registry explicitly says which model to use, trust that as the primary routing signal** rather than env-var heuristics.

Implementation:
```rust
fn build(model_override: Option<&str>) -> Result<Self> {
    // NEW: Model-aware routing takes priority
    if let Some(model) = model_override {
        // 1. Check MODEL_PROVIDER_MAP first
        if let Some(provider) = resolve_provider_for_model(model) {
            match provider.as_str() {
                "fireworks" if FireworksClient::is_configured() =>
                    return Self::build_fireworks_chain(Some(model)),
                "openai" if has_api_key_for_provider("openai") =>
                    return Self::build_openai_direct_chain(model),
                "anthropic" if has_api_key_for_provider("anthropic") && !proxy_is_configured() =>
                    return Self::build_anthropic_direct_chain(model),
                _ => {} // Fall through to env-based routing
            }
        }
        // 2. Auto-detect Fireworks models by prefix (even without MODEL_PROVIDER_MAP)
        if model.starts_with("accounts/fireworks/") && FireworksClient::is_configured() {
            info!(model, "Auto-detected Fireworks model from prefix — using FireworksClient directly");
            return Self::build_fireworks_chain(Some(model));
        }
    }

    // Existing env-based routing (proxy → fireworks → direct)
    let proxy_active = proxy_is_configured();
    let fireworks_active = FireworksClient::is_configured();
    // ... rest unchanged
}
```

### 1.2 Add `build_openai_direct_chain` and `build_anthropic_direct_chain`

**File**: `crates/agent-client/src/fallback.rs`

These are simple single-provider chains (no fallbacks needed when model is explicitly routed):

```rust
fn build_openai_direct_chain(model: &str) -> Result<Self> {
    let mut clients: Vec<Box<dyn LlmClient>> = Vec::new();
    if let Ok(c) = OpenAiClient::from_env_with_model(model) {
        info!(provider = "openai-direct", model = %c.model(), "Direct OpenAI client initialized");
        clients.push(Box::new(c));
    }
    if clients.is_empty() { bail!("OpenAI direct mode: OPENAI_API_KEY not set") }
    let timeout_secs = std::env::var("LLM_TIMEOUT_SECS").ok().and_then(|s| s.parse().ok()).unwrap_or(60);
    Ok(Self::new(clients, Duration::from_secs(timeout_secs)))
}

fn build_anthropic_direct_chain(model: &str) -> Result<Self> {
    let mut clients: Vec<Box<dyn LlmClient>> = Vec::new();
    if let Ok(c) = AnthropicClient::from_env_with_model(model) {
        info!(provider = "anthropic-direct", model = %c.model(), "Direct Anthropic client initialized");
        clients.push(Box::new(c));
    }
    if clients.is_empty() { bail!("Anthropic direct mode: ANTHROPIC_API_KEY not set") }
    let timeout_secs = std::env::var("LLM_TIMEOUT_SECS").ok().and_then(|s| s.parse().ok()).unwrap_or(60);
    Ok(Self::new(clients, Duration::from_secs(timeout_secs)))
}
```

### 5.2 Add "Why Codex is Simpler" section to README

Add a new section explaining the key difference:

```markdown
## Why Codex Doesn't Need a Proxy

| | Codex CLI | Claude Code CLI |
|---|---|---|
| **API Format** | OpenAI Chat Completions | Anthropic Messages |
| **Third-party providers** | Direct connection | Requires proxy |
| **Env vars** | `OPENAI_API_KEY` + `OPENAI_BASE_URL` | `ANTHROPIC_API_KEY` + `ANTHROPIC_BASE_URL` |
| **Fireworks** | Set `OPENAI_BASE_URL=https://api.fireworks.ai/inference/v1` | Must run `anthropic-mock` proxy |

**Codex CLI speaks the OpenAI Chat Completions format** (`/v1/chat/completions`), which is the industry-standard interchange format. Nearly every third-party provider (Fireworks, DeepSeek, Together, Groq, etc.) supports this format natively. This means Codex can connect directly to any of them with just `OPENAI_BASE_URL` and `OPENAI_API_KEY`.

**Claude Code CLI speaks the Anthropic Messages format** (`/v1/messages`), which only Anthropic's own API supports natively. When you want to route Claude Code through a non-Anthropic provider, you need the `anthropic-mock` proxy running locally to translate Anthropic → OpenAI format in real-time.
```

### Claude Code CLI (Anthropic Format — Proxy Required for Third-Party Providers)

| Provider | Proxy Required? | How |
|----------|----------------|-----|
| Anthropic (direct key) | No | Just set `ANTHROPIC_API_KEY` |
